**보강 데이터 (같은 날 별도 분석, `vault work/프로젝트/260707-분석-코리너스-랜딩-CTA전환-개선.md`):** 최근 3일 실측 기준 방문 97%가 모바일 인앱 웹뷰(iOS 인스타그램/안드로이드 페이스북), 방문자 99% 한국(일본 진출 검토하는 한국 브랜드 담당자), 평균 스크롤 33%, 유일한 문의 폼은 페이지 96% 깊이. 인앱 웹뷰는 주소창/북마크가 없어 화면에 보이는 버튼이 유일한 액션 경로 — 상시 노출 CTA 와 페이지 이동 없는 폼(모달/시트)이 §3 의 핵심 근거로 추가됨. 구현 계획 Phase 3 이 이를 반영.

**전략 결론 3줄:**
1. **1순위 = 모바일 홈 첫 1.5스크린.** 92%의 방문자에게 사이트란 이 영역이 전부다. 여기서 "누구인지, 뭘 해주는지, 왜 믿을지"가 6초 안에 전달돼야 한다.
2. **2순위 = /contact 폼 경험.** 유일한 전환 지점이고 트래픽 2위.
3. **3순위 = 데스크톱 풀 내러티브.** PC 방문자(스크롤 66%, 139초)는 검토 모드의 잠재 고객/파트너 — 지금의 긴 스토리텔링이 유효한 대상. 데스크톱은 다듬고, 모바일은 재구성한다.

## 1. 전역 개선 — 디자인 시스템 레벨

현행 Organic Warm v2 (`design-system/MASTER.md`) 유지 전제. 아래는 시스템의 빈 곳을 채우는 작업이다.

### 1-1. 타이포그래피: KR 헤드라인 격차 해소
- 현상: EN 디스플레이는 Barlow Condensed 700의 드라마가 있는데, KR 헤드라인은 Noto Sans KR bold 그대로라 평이하다. 사이트 카피의 대부분이 KR이므로 체감 품질을 KR 쪽이 결정한다.
- 개선: KR 디스플레이 규격을 시스템에 추가 — `letter-spacing: -0.03em`, `line-height: 1.05~1.1`, 사이즈 스케일 한 단계 상향, 헤드라인 내 핵심 구절만 accent 컬러/weight 대비(현행 portfolio 히어로의 2줄 대비 패턴을 전 페이지 규격으로).
- 선택지(브랜드 결정 필요): KR 전용 디스플레이 폰트 도입(예: Pretendard 800~900을 헤드라인 한정 사용). 기본안은 폰트 추가 없이 Noto Sans KR 튜닝.
- 부수: `globals.css` body 폴백 스택의 'Inter'(실제 미로드) 정리, SEO Batch D 의 LCP font-swap 항목 동시 처리.

### 1-2. 컬러/표면: 다크의 깊이
- 현상: 다크 섹션이 전부 단일 플랫 `#1C1917`이라 섹션 간, 카드 간 깊이 위계가 없다.
- 개선: elevation 토큰 2단 추가 (`--surface-1: #232019` 급, `--surface-2: #2A2620` 급 + 보더 `rgba(168,162,158,0.15)` 현행 유지). 카드, 폼 필드, nav 스크롤 상태에 적용. 새 hue 도입은 금지(오렌지 절제 원칙 유지).
- 크림 섹션에는 미세 정적 noise 텍스처(CSS, 2~3% 불투명) 검토 — 셰이더 레이어와 질감 톤 통일.

### 1-3. 모션 시스템: 어휘 정리 + 빈화면 제거
- 현상: whileInView fade-up 단일 어휘의 반복. 빠른 스크롤 시 섹션이 빈 상태로 먼저 보이는 순간이 있음.
- 개선:
  - 모션 어휘 3종으로 위계화: 섹션 진입 reveal(현행), 리스트 stagger(현행), 스탯 숫자 count-up(신규 — Results, About 스탯, contact 스탯 스트립).
  - `viewport={{ margin: '-15% 0px' }}` 계열로 트리거 선행, `once: true` 유지.
  - 마이크로 인터랙션 규격: 버튼 press(scale 0.98), 폼 focus 링(accent 1px + soft glow), nav 링크 밑줄 슬라이드(현행 유지), 카드 hover(현행 translateY 유지).
  - 모바일 sticky 문의 CTA: 홈/서비스/포트폴리오 스크롤 30% 이후 하단 고정 바(전환 동선 — 92% 모바일에서 nav 상단 CTA는 손가락 동선 밖).

### 1-4. 네비게이션/푸터
- nav: 스크롤 시 높이 축소 + backdrop-blur + `--surface-1` 배경 전환. 모바일 햄버거 메뉴 열림 상태의 시각 품질 점검(현재 기본적).
- 푸터: 현재 등기 정보 3줄뿐. 사이트맵 링크(6페이지), sales@koreaners.com, 인스타그램 2계정, 문의 CTA 미니 밴드를 갖춘 표준 푸터로 확장. 내부 링크 구조 개선(SEO 겸용).

### 1-5. 품질/접근성/부채
- 대비: 다크 위 회색 서브카피(`text-muted` 계열) WCAG AA 검증, 미달분 한 단계 상향.
- dead clicks 대응: 클릭 안 되는 pill/카드에 `cursor: default` 명시 + 실제로 이동 가능한 것(로고월의 포트폴리오 보유 브랜드, 포트폴리오 카드 전체 영역)은 클릭 영역 확대.
- 데드 코드: `.dark` oklch 블록(미사용 shadcn 잔재), `tailwindcss-animate` vs `tw-animate-css` 중복 정리.
- 포커스 비저블 스타일 전역 규격(키보드 접근).

## 2. 홈 개선 (트래픽 78%, 최우선)

### 2-1. 모바일 첫 1.5스크린 재설계 — 이번 작업의 핵심
현상: 첫 화면이 "BEYOND / AGENCY / 코리너스 / CTA 2개"로 끝. 브랜드는 강하지만 **무엇을 해주는 회사인지, 왜 믿을지가 첫 화면에 없다.** 모바일 방문자의 86%가 여기서 끝낸다(스크롤 13.9%).

개선 스펙:
- 히어로 구성 유지 + 보강: BEYOND/AGENCY 타이포는 시그니처로 유지하되, 그 아래 '코리너스' 한 단어 대신 **KR 한 줄 가치제안** ("일본 진출 크리에이터 마케팅, 진단부터 성과까지" 급 — 카피는 Leo 결정)과 **신뢰 신호 1개**(수출바우처 공식 수행기관 pill 또는 "185개 브랜드" 한 줄)를 넣는다.
- 히어로 직하단(스크롤 1화면째)에 로고월 축약판 또는 3대 스탯을 배치해 신뢰를 선행시킨다. 현재 스탯(Results)과 로고월은 페이지 하단 — 92%가 못 보는 위치다.
- 모바일 정보 순서 재배치: 히어로 → 신뢰(스탯+로고) → 포트폴리오 3장 → CTA → (이하 barriers/roadmap 장문은 후반). 데스크톱은 현행 내러티브 순서 유지 가능 — 순서 분기가 구현상 과하면 공통으로 신뢰 블록만 위로 올린다.
- 배경: 셰이더 레이어(§5) — 모바일에서도 히어로가 유일한 노출 영역이므로 모바일 히어로 품질에 투자 가치가 가장 크다.

### 2-2. 페이지 리듬 압축
- 홈 전체 7,640px. Solution Roadmap 01~04 구간이 초대형 타이포 + 우측 대공백으로 체감 길이의 절반을 차지. 수직 패딩 30~40% 압축, 스텝별 우측에 소형 시각 요소(키워드 pill 재배치 또는 미니 다이어그램) 또는 좌우 교차 레이아웃.
- 목표 전체 길이 ~5,500px (데스크톱 기준), 모바일은 섹션 축약형.

### 2-3. 마이크로 개선
- Results 스탯 count-up + gradient-warm-text 유지.
- Market Opportunity 스탯 카드: 셰이더 업그레이드 대상(§5), 카드 자체는 유지.
- 포트폴리오 카드 3장: hover 시 이미지 scale(현행) + 카드 전체 클릭 영역 확인.

## 3. /contact 개선 (전환 페이지, 트래픽 2위)

광고 유입 전환 지점이므로 **배포는 보수적으로, 측정 가능하게.** 수치/카피는 `meta-ads-automation/config/verified_numbers.json` 화이트리스트 게이트 준수(변경 금지).

- 폼 UX: 현재 필드 5개 + 체크박스 2개가 한 번에 노출 — 모바일에서 길다. 개선안 (a) 필수 필드 시각 위계 강화 + 선택 필드 축소, (b) 2-step 분리(1단계 이름/연락처, 2단계 상세) 중 택1. b안은 전환 실험 성격이므로 픽셀 이벤트 유지 확인 후 적용.
- 입력 중 인라인 유효성(전화번호 형식 등 현재 헬퍼 텍스트만 있음), 제출 중/성공/실패 상태의 명확한 피드백.
- 폼 주변 신뢰 요소: 수행기관 pill(현행 유지) + 대표 브랜드 로고 3개를 폼 카드 안 또는 직상단에.
- 모바일에서 폼 도달 전 스크롤 거리 최소화(히어로 CTA가 폼 앵커로 스크롤 — 현행 유지, 거리 단축).

## 4. 서브페이지 개선

### 4-1. Service (83세션)
- 텍스트 월 완화: CHALLENGES/프로세스 구간에 다이어그램 1개(4단계 프로세스 타임라인 시각화). 새 일러스트 시스템을 만들지 말고 기존 pill + 라인 + 넘버 어휘로 조립.
- 섹션 리듬은 홈 §2-2와 같은 원칙으로 압축.

### 4-2. Creator (106세션)
- 현상: 크리에이터 카드 그리드가 동의 문제로 숨김(`SHOW_CREATOR_CARDS = false`) — 페이지가 빈약하고 "200명+" 주장의 시각 증거가 없다.
- 개선: 동의 확보 전 대안으로 익명화 통계 카드 그리드(티어 분포, 카테고리 분포, 플랫폼 분포 — 실데이터 기반) 또는 블러 처리 실루엣 그리드. 합류 폼(Dialog)은 모바일 풀스크린 시트로.

### 4-3. Portfolio + 상세 (176 + 상세 다수)
- 인덱스: 카드 썸네일 톤 불균일(로고 이미지/사진/흑백 혼재) — 썸네일 오버레이 규격(하단 그라데이션 + 카테고리 pill 위치 고정)으로 톤 통일.
- 상세: 히어로 이미지 아래 KPI 스탯 밴드(조회수/전환 등 검증 수치), 본문 하단 관련 사례 2~3장 + 문의 CTA 밴드. 상세 페이지가 광고/영업에서 직접 링크되는 URL이므로 단독 랜딩으로도 성립하게.

### 4-4. Blog + 상세 (46세션, GEO 자산)
- 인덱스: 동일 로고 썸네일 반복 제거 — 카테고리별 셰이더 정적 아트 에셋(§5) 또는 `/api/og` 확장.
- 상세: 다크 배경 장문 리딩은 피로 — 본문 카드만 크림(`#FAF7F2`) 표면으로 반전(다크/크림 교차라는 브랜드 리듬과 정합). 목차(3개 이상 헤딩 시), 읽기 진행 바, 본문 폭 `65ch` 상한 + 행간 1.8 확인.
- 트래픽은 작지만 GEO/SEO 파이프라인의 얼굴이므로 리딩 품질 우선.

### 4-5. About / Careers / JP 로케일
- about: 스탯 아이콘+숫자 블록 정렬 정돈, 서사 1~2문단 보강(경영진/연혁은 별도 결정).
- careers: 현행 시스템 유지, JobPosting 스키마 살아있는지 확인만.
- JP: 히어로 서브카피 "コリアナース" 표기 검수 필요(코리너스 가타카나 표기 확정 — 현재 표기는 "코리아 널스"로 읽힐 여지). JP 폰트(Noto Sans JP) 렌더는 정상.

## 5. 셰이더 레이어 (별첨 문서로 집행)

상세 스펙, 파라미터, 가드레일 10조, 페이즈는 `docs/plans/2026-07-07-frontend-shader-design-guide.md` 참조. 본 가이드 관점의 요지만:

- 컨셉: "hero-glow 를 살아있는 빛으로" — 리디자인이 아니라 재질 업그레이드.
- Clarity 반영 수정: 모바일이 92%이고 노출이 히어로에 집중되므로, **모바일 히어로 셰이더 품질이 데스크톱보다 중요하다.** 별첨 §3-8의 "성능 회귀 시 모바일 정적 프레임 강등" 원칙은 유지하되, 기본값은 모바일에서도 저속 애니메이션(`minPixelRatio 1` + `speed 0.15`)으로 시작하고 실기기 측정으로 판단.
- 적용 순서도 트래픽 따라 조정: 홈 히어로 → /contact 정적 프레임 검토(전환 페이지라 보수적, 별첨 Phase 4 → **Phase 2로 상향 검토**, 단 A/B 또는 최소한 전환율 전후 비교 가능할 때만) → 서브 히어로 → 블로그 에셋.

## 6. 실행 로드맵 (임팩트 x 트래픽 기준)

| Sprint | 범위 | 게이트 |

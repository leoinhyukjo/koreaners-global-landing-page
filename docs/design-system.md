## 1. Color Tokens

```css
--kn-base:    #141414;    /* 배경 */
--kn-accent:  #FF4500;    /* 강조 — 유일한 컬러 */
--kn-text:    #FFFFFF;    /* 텍스트 */
--kn-muted:   #A0A0A0;    /* 보조 텍스트 (날짜, 캡션) */
--kn-border:  #FF450033;  /* 액센트 20% 투명 (카드 보더) */
```

### Tailwind 확장

```js
// tailwind.config
colors: {
  kn: {
    base:   '#141414',
    accent: '#FF4500',
    text:   '#FFFFFF',
    muted:  '#A0A0A0',
  }
}
```

### 규칙

- 이 5개 외 색상 사용 금지
- 사진을 제외한 모든 UI 요소는 이 팔레트 안에서
- 그라데이션 사용 금지 — 플랫 컬러만
- "이거 써도 되나?" 싶으면 쓰지 않는 게 맞음

## 2. Typography

| 역할 | 폰트 | Weight | 용도 |
|------|-------|--------|------|
| **Display** | Black Han Sans | 400 (기본이 극굵) | 썸네일 헤드라인, 히어로 타이틀 |
| **Body KR** | Pretendard | 400 / 600 | 본문, UI 텍스트 |
| **Body EN** | Geist | 400 / 600 | 영문 본문 |
| **Mono** | Geist Mono | 400 | 코드, 숫자 강조 |

### 타이포 문법

- 헤드라인은 **최대한 크게, 2-3줄**
- 키워드 하나를 **이탤릭** 또는 **액센트 컬러(#FF4500)**로 강조
- 본문은 작게, 절제하여
- 영문은 ALL CAPS 가능, 한글은 폰트 크기로 위계 표현

### 폰트 사이즈 스케일

```
Display:  48-72px (썸네일/히어로)
H1:       36-48px
H2:       28-36px
H3:       20-24px
Body:     15-16px
Caption:  12-13px
```

## 3. 썸네일 템플릿 시스템

### 사이즈 규격

| 용도 | 사이즈 | 비율 |
|------|--------|------|
| OG / 블로그 | 1200 x 630 | 약 2:1 |
| SNS 정사각 | 1080 x 1080 | 1:1 |
| 포트폴리오 카드 | 1200 x 800 | 3:2 |

### 레이아웃

```
┌──────────────────────────────────┐
│ [KN 로고]              [카테고리]│  ← 상단 바
│                                  │
│                                  │
│   헤드라인 첫째줄                 │  ← Black Han Sans, #FFFFFF
│   *강조 단어*                    │  ← #FF4500
│   헤드라인 셋째줄                 │
│                                  │
│  ──────────── #FF4500 ──────────│  ← 하단 액센트 라인
└──────────────────────────────────┘
배경: #141414
```

### 변형 3종

| 변형 | 구성 | 용도 |
|------|------|------|
| **A. 타이포 온리** | 다크 배경 + 텍스트만 | 블로그 기본 |
| **B. 사진 + 타이포** | 좌측 텍스트 + 우측 인물/제품 사진 | 포트폴리오, 사례 |
| **C. 액센트 풀배경** | #FF4500 배경 + #FFFFFF 텍스트 | 특별 콘텐츠, CTA |

### 썸네일 체크리스트

- [ ] 로고 좌상단에 있는가
- [ ] 헤드라인 2-3줄, Black Han Sans인가
- [ ] 강조 단어 1개가 #FF4500인가
- [ ] 배경은 #141414 또는 #FF4500 둘 중 하나인가
- [ ] 색상이 5개 토큰 안에 있는가

## 5. 사진 디렉션

### 톤

- 따뜻한 톤 보정 (오렌지 액센트와 매칭)
- 채도 약간 낮추고, 따뜻한 색온도로

### 주제

- 일본 스트리트, 도쿄 감성
- 크리에이터 촬영 현장
- 제품/브랜드 비주얼
- 한일 문화 크로스오버

### 처리

- 다크 오버레이: `rgba(20, 20, 20, 0.4)` — 텍스트 가독성 확보
- 사진 위에 텍스트 올릴 때 반드시 오버레이 적용

### 금지

- 차가운 블루톤
- 스톡사진 느낌 (어색한 포즈, 과도한 밝기)
- 복잡한 배경 위에 오버레이 없이 텍스트

## 6. 섹션 리듬 (웹사이트)

다크/액센트 교대로 시각적 호흡을 만든다.

```
[#141414] 히어로 — 타이포 중심, 마퀴 텍스트
[#FF4500] 가치제안 / 소개 — 흰색 텍스트
[#141414] 서비스 상세
[#FF4500] 실적 / 숫자 하이라이트
[#141414] 포트폴리오 그리드
[#141414] 블로그 최신글
[#FF4500] CTA — "함께 일하기"
[#141414] 푸터
```

### 섹션 규칙

- 액센트 섹션은 전체의 30% 이하 (과하면 피로)
- 다크 섹션이 기본, 액센트는 강조 포인트
- 섹션 전환 시 마퀴 텍스트 또는 1px 구분선

## 7. 컴포넌트 스타일

### 버튼

```css
/* Primary */
background: #FF4500;
color: #FFFFFF;
border: none;
font-family: 'Pretendard', sans-serif;
font-weight: 600;

/* Secondary */
background: transparent;
color: #FFFFFF;
border: 2px solid #FF4500;

/* Ghost */
background: transparent;
color: #FFFFFF;
border: 2px solid #FFFFFF;
```

### 카드

```css
background: #141414;
border: 1px solid rgba(255, 69, 0, 0.2);  /* --kn-border */
border-radius: 0;  /* sharp corners */

/* Hover */
border-color: #FF4500;
transform: translateY(-2px);
transition: all 0.2s;
```

### CTA 링크 (FORMA 스타일)

```
텍스트 + 화살표 ➺
예: "자세히 보기 ➺"  "문의하기 ➺"
버튼 형태가 아닌 텍스트 링크로
```

## 8. 로고 사용 가이드

- **다크 배경**: 흰색 버전 (현재 로고 반전)
- **액센트 배경**: 흰색 버전
- **라이트 배경**: 원본 블랙 버전
- **최소 여백**: 로고 높이의 50%
- **최소 크기**: 높이 24px

## 9. Do's and Don'ts

### Do's ✓

- 큰 타이포로 임팩트 주기
- 여백을 넉넉히 — 빈 공간이 곧 고급감
- 하나의 강조 단어에 액센트 컬러
- 사진은 따뜻한 톤으로 통일
- sharp corners (border-radius: 0)

### Don'ts ✗

- 3개 이상의 색상 사용
- 그라데이션, 글로우, 네온 효과
- 3D 일러스트, 아이콘 남발
- 둥근 모서리 (pills, rounded cards)
- 작은 텍스트로 많은 정보 넣기
- 장식 목적의 아이콘/일러스트
